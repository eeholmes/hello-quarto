A common first question is “how are R Markdown and Quarto different, under the hood, particularly given the rename of the file just works?” The main difference is that with R Markdown, the R package rmarkdown does the heavy lifting of the orchestration of going from source code (the Rmd file) to output, using knitr for evaluating the code chunks. With Quarto, the Quarto command line interface (or CLI, for short) does the orchestration of processing executable code chunks with either knitr or jupyter and then converting the resulting markdown file to the desired output. While this is technically impressive, I’ll be honest, it’s not necessarily what sparked my interest in Quarto in the first place.
